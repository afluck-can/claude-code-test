# Landing Page Redesign Skill

## Purpose
This skill helps upgrade and modernize landing page UI/UX while preserving performance, accessibility, and brand consistency. It analyzes existing code and design structure, proposes improved layout concepts, and implements production-ready frontend updates.

The focus is on modern SaaS landing page standards, conversion optimization, and clean frontend implementation.

---

# When to Use This Skill

Use this skill when:

- Improving the visual design of a landing page
- Modernizing an outdated UI
- Increasing conversion and clarity of value proposition
- Refactoring landing page components
- Improving responsive layout and accessibility
- Implementing modern frontend design patterns

Typical triggers include:

- "Improve the landing page"
- "Modernize this UI"
- "Redesign the homepage"
- "Increase landing page conversion"
- "Update the UI while keeping the same stack"

---

# Core Responsibilities

The agent using this skill acts as both:

- **Senior Product Designer**
- **Senior Frontend Engineer**

It must improve both **design quality and implementation quality**.

---

# Primary Objectives

1. Upgrade landing page aesthetics and usability.
2. Apply modern design principles including:
   - visual hierarchy
   - typography systems
   - spacing and layout rhythm
   - responsive design
   - accessibility best practices
3. Maintain strong performance and lightweight code.
4. Preserve brand identity and existing design system constraints.
5. Implement improvements using the existing tech stack.

---

# Preferred Technology Stack

Prioritize working within the repository's current stack.

Common expected technologies include:

- Next.js
- React
- Tailwind CSS
- shadcn/ui

Avoid introducing unnecessary dependencies.

---

# Workflow

## 1. Inspect the Codebase

Locate landing page related files and components.

Typical locations:

- `pages/`
- `app/`
- `components/`
- `sections/`
- `layouts/`

Review:

- component hierarchy
- layout patterns
- styling approach
- responsiveness
- accessibility practices

Also review:

- `CLAUDE.md`
- `README`
- design system documentation
- Tailwind config
- theme or token files

---

## 2. Audit the Current Landing Page

Evaluate the page for:

- visual hierarchy
- whitespace and spacing consistency
- typography scale
- clarity of value proposition
- CTA prominence
- trust signals
- section structure
- responsiveness
- accessibility
- code maintainability

Produce a concise audit identifying:

- strengths
- weaknesses
- biggest improvement opportunities

---

## 3. Generate Concept Variations

For each major landing page section, propose **at least three design concepts**.

Typical sections include:

- hero
- features
- testimonials / social proof
- CTAs
- navigation
- footer

Each concept should describe:

- layout structure
- visual tone
- user experience improvement
- potential conversion benefit
- estimated implementation complexity

---

## 4. Recommend a Direction

Select the best concept direction based on:

- clarity
- conversion potential
- brand alignment
- implementation simplicity
- maintainability

Explain the reasoning briefly.

---

## 5. Implement the Improvements

Update the code to implement the selected design.

The implementation must:

- follow existing repo conventions
- be modular and maintainable
- work across breakpoints
- maintain accessibility standards
- avoid unnecessary dependencies

Code should be ready for direct repository integration.

---

# Design Principles to Apply

Focus on improving:

- visual hierarchy
- whitespace rhythm
- section separation
- typography scale
- CTA visibility
- content scannability
- credibility and trust

Preferred modern patterns include:

- card-based feature sections
- refined spacing systems
- subtle shadows
- strong hero structure
- lightweight motion and hover states
- tasteful gradients only if brand appropriate

Avoid:

- cluttered layouts
- heavy animation
- poor contrast
- excessive visual noise
- unnecessary UI complexity

---

# Accessibility Requirements

Ensure:

- semantic HTML structure
- keyboard accessibility
- visible focus states
- sufficient color contrast
- screen reader compatibility
- responsive design across devices

Follow WCAG accessibility guidelines where possible.

---

# Performance Requirements

Maintain fast load times by:

- optimizing images and assets
- minimizing large dependencies
- avoiding excessive animations
- ensuring efficient component rendering

---

# Allowed File Modifications

The agent may edit landing page related files including:

- `.js`
- `.jsx`
- `.ts`
- `.tsx`
- `.css`
- UI component files
- page/layout files
- theme or styling configuration

---

# Output Expectations

When this skill is used, the agent should produce:

1. **Landing Page Audit**
2. **Concept Design Options**
3. **Recommended Direction**
4. **Production-Ready Code Updates**
5. **Summary of Improvements**
6. **Optional A/B Testing Ideas**

Explanations should be concise and focused on practical value.

---

# Design Inspiration (Optional)

When needed, reference high-quality SaaS design patterns and best practices to guide layout decisions.

Examples may include:

- modern hero sections
- feature comparison layouts
- social proof placement
- strong call-to-action patterns

---

# Guiding Principle

Preserve what works, modernize what does not, and ensure every change improves **clarity, usability, or conversion potential**.
